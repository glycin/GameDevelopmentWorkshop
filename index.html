Game Development Workshop

The Editor

Here you can find information regarding the Unity editor and its layout. Every tab is draggable and rearangable, so you can design the editor to your liking. The Unity Editor is fully extendible, and someday you might even build your very own editor scripts!

The Scene View

The scene view is the main view for developming within Unity. Every object you create is visible in the scene view. Using the moving tools you can position, rotate and scale the selected objects.Using the mouse you can rotate and reposition the camera, allowing you to get a clear view of everything that is happening in you scene!

The Game View

The game view actually shows how the game will show during run time. The game you are creating is fully playable and observable in the game view. That means you can quickly test your changes just by pressing the Play button.

The Hierarchy Window

In the Hierarchy window you can find a list of all the current objects in your scene. You can use this window to quickly search and find a certain object. Furthermore this functions alot like a file explorer. You can drag and drop objects under other objects in order to parent them. Parented objects inherit the position, scale and rotation of the parent. This window is also the place where you can delete unwanted objects or create new ones.

The Project Window

The project window shows everything that is within your Assets folder. It's completely identical to the structure you see within the file explorer. Everything you change within the file explorer will be reflected in the project window and vice-versa. Try to keep your assets organized, since you will be needing them to create your game, and finding them quickly is essential.

The Inspector Window

In the Inspector window you can view and modify the components of the selected game objects. If the objects share the same components, you can edit the values of the components simultaneously. You can drag and drop scripts in the Inspector to add them to the selected game objects. You can also press the add component button and select the component you need. All object modification happens here in the inspector window. You can change the name, tag, layer, position, rotation, scale, material etc. of the object.

The debugging console

The console window is the output window. Errors, exceptions, warnings and logs are shown here during runtime. Double click the message and the IDE of your choosing will open up and jump to the line that produced the message. You can log messages using the following code:


            // Using the Debug static class: https://docs.unity3d.com/ScriptReference/Debug.html
            Debug.Log("The name of this gameObject is: " + gameObject.name);
            //Or
            print("the name of this gameObject is: " + gameObject.name);

Tags and Layers

Tags and Layers allow us to define some common functionality across unrelated game objects. For example some physics operations can ignore objects in certain layers. Or some triggers will react only to objects with a certain tag. Using this you can easily group objects wich should react in a similar way to certain actions. To define new tags and layers you click on an object and in the Inspector you click on the drop down menu.

Once the drop down menu appears, you click on "Add Tag" or "Add Layer" and then press the "+" icon to add a new tag or layer
