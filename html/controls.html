Controls

Controls are the only way for the user to interact with your application. Having smooth and responsive controls ensures a better user experience.

Input

The main way to handle keyboard and mouse input in Unity is through the Input class.
It provides a way to handle input either by Key or by Axis.

There are 3 methods for Key input:

  • GetKeyDown(KeyCode key): Whether a key was just pressed (once).
  • GetKeyUp(KeyCode key): Whether a key was just released (once).
  • GetKey(KeyCode key): Whether a key is being pressed now (continuous).
    private void Update() {
        if (Input.GetKey(KeyCode.W)) { // Fires constantly
            // Move forward
        }

        if (Input.GetMouseButton(0)) { // Fires once
            // Shoot
        }
    }
Hardcoding a specific keycode makes the controls less configurable. Unity offers a built-in solution for that already, called Axes, which they can be easily configured either from Unity or in runtime.
    private void Update() {
        if(Input.GetAxis("Vertical") > 0) { // More configurable
            var amount = Input.GetAxis("Vertical");
            // Move forward by amount [0-1]
        }
    }
The axis settings can be found under Input settings (Edit -> Project Settings -> Input)
